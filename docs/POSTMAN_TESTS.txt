POSTMAN TESTS
==============

Base URL
http://localhost:5000

Note
- In this project, routes are mounted directly on `/`, not `/api`.
- Use `Content-Type: application/json` for all POST requests.
- Protected routes need `Authorization: Bearer <TOKEN>`.

Suggested test order
1. Health check
2. Signup
3. Signin
4. Profile
5. Admin-only get all users

1. Health Check
---------------
Method: GET
URL: http://localhost:5000/

Expected:
{
  "message": "Cinema Booking API is running"
}


2. Signup
---------
Method: POST
URL: http://localhost:5000/users/signup

Body:
{
  "fullName": "eslam gamal",
  "email": "ahmed@example.com",
  "password": "123456",
  "phoneNumber": "01012345678",
  "dateOfBirth": "2000-05-15"
}

Expected:
- returns `success: true`
- returns `data.user`
- returns `data.token`

Save:
- user token as `USER_TOKEN`
- user id as `USER_ID`


3. Signin

}

Expected:

4. Login With Google
--------------------
Method: POST
URL: http://localhost:5000/users/google

Body:
{
  "idToken": "PUT_GOOGLE_ID_TOKEN_HERE"
}

Expected:
- returns `success: true`
- creates a new google user if email does not exist
- returns `data.token`

Required env:
- GOOGLE_CLIENT_ID
- JWT_SECRET

7. Create Hall
--------------
Method: POST
URL: http://localhost:5000/halls

Headers:
Authorization: Bearer <ADMIN_TOKEN>

Body:
{
  "name": "Hall 1",
  "rows": 5,
  "cols": 6,
  "seatLayout": [
    { "row": "A", "type": "standard" },
    { "row": "B", "type": "standard" },
    { "row": "C", "type": "premium" },
    { "row": "D", "type": "premium" },
    { "row": "E", "type": "vip" }
  ]
}

Expected:
- returns created hall

Save:
- hall id as `HALL_ID`

------------------
Method: GET
URL: http://localhost:5000/halls/overview

Useful Postman variables
------------------------
- baseUrl = http://localhost:5000
- userToken = token from signup/signin
- adminToken = token from admin account
- userId = id from signup response
- hallId = id from create hall response
- movieId = id from create movie response
- showtimeId = id from create showtime response
- bookingId = id from create booking response
