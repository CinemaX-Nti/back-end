POSTMAN TESTS
==============

Base URL
http://localhost:5000

Note
- In this project, routes are mounted directly on `/`, not `/api`.
- Use `Content-Type: application/json` for all POST requests.
- Protected routes need `Authorization: Bearer <TOKEN>`.

1. Health Check
---------------
Method: GET
URL: http://localhost:5000/

Expected:
{
  "message": "Cinema Booking API is running"
}


2. Signup
---------
Method: POST
URL: http://localhost:5000/users/signup

Body:
{
  "firstName": "eslam",
  "lastName": "gamal",
  "email": "ahmed@example.com",
  "password": "123456",
  "phoneNumber": "01012345678",
  "dateOfBirth": "2000-05-15"
}

Expected:
- returns `success: true`
- returns `data.user`
- returns `data.requiresEmailConfirmation: true`
- sends a confirmation OTP email

Save:
- user id as `USER_ID`
- confirmation OTP from email inbox as `CONFIRMATION_OTP`

7. Update Password
------------------
Method: PATCH
URL: http://localhost:5000/users/update-password

Headers:
Authorization: Bearer <USER_TOKEN>

Body:
{
  "currentPassword": "123456",
  "newPassword": "12345678"
}

Expected:
- status 200
- returns `success: true`
- `message: "Password updated successfully"`

Notes:
- works only for local accounts
- use the new password in the next signin test


8. Forget Password
------------------
Method: POST
URL: http://localhost:5000/users/forget-password

Body:

  "email": "ahmed@example.com"
}

Expected:

- returns a generic success message even if the email does not exist

Useful Postman variables
------------------------
- baseUrl = http://localhost:5000
- userToken = token from signup/signin
- adminToken = token from admin account
- userId = id from signup response
- confirmationOtp = OTP from email confirmation message
- resetOtp = OTP from password reset email
- deleteProfileOtp = OTP from profile deletion email
- hallId = id from create hall response
- movieId = id from create movie response
- showtimeId = id from create showtime response
- bookingId = id from create booking response
